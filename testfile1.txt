git init "NameDesGitOrdners" //erstellt einen Git Ordner, enthält die Histrie-diles und zu bearbeitenden files. 
git add "Dateiname" // nicht permanent, sagt, dass die Datei im nächsten Commit dabei ist
git commit -m"m nutzt man um einen Kommentar in diesen Gänsefüßchen zu schreiben" // erstellt einen Snapshot von der aktuellen Dateiversion. Danach KEINEN Dateinamen hinschreiben. 
git checkout master //holt die aktuelle Master version.
git status //zeigt an, welche Dateien noch "uncommited" sind
git rm datei.txt //entfernt Datei
git diff, zeigt an, ob nicht commitete änderungen vorhanden sind. grün: das was hinzukam, rot was entfernt wurde.
Will man eine Datei löschen, löscht man sie ganz normal im Editor der Datei selbst, dann wieder git add "Dateiname" und dann git commit -m"..." bestätigt das löschen. oder besser man benutzt: git rm dateiname.txt
